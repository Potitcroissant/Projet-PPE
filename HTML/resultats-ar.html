Arabe

Dans le cas de l’arabe, afin d’analyser les contextes dans lesquels le mot نظر (“regard / point de vue”) est utilisé, nous avons aussi utilisé le script PALS et suivi les mêmes démarches que l’anglais et le français. Pour éviter les répétitions, nous expliquons ici les adaptations nécessaires pour le traitement de l’arabe et les principaux résultats obtenus.

Le code pour la tokénisation est donc le suivant :

rm -rf corpus-texte/ar
mkdir -p corpus-texte/ar

lineno=1
for f in dump-text/ar-*.txt
do
  sed '1,/^$/d' "$f" | sed -E 's/([^[:space:][:punct:]]+|[[:punct:]])/\\1\\n/g' | grep -v '^[[:space:]]*$' > corpus-texte/ar/ar-$lineno.txt
  lineno=$((lineno+1))
done

Pour l’arabe, nous retirons d’abord l’entête HTTP des fichiers dump (avec sed '1,/^$/d') afin de limiter le bruit avant la tokenisation.

Ensuite, nous appliquons le script PALS (adaptation des chemins et du mot cible) :

python3 PALS/cooccurrents.py --target نظر --match-mode exact corpus-texte/ar/* > resultat-ar.csv

Le terminal indique que le mot cible نظر apparaît 82 fois. Le fichier resultat-ar.csv contient 760 lignes (cooccurrents).

En analysant les cooccurrences, on se rend compte que نظر apparaît dans plusieurs contextes :

  • celui du point de vue, avec des formes comme وجهة / وجهات (souvent dans « وجهة نظر »)
  • celui de la vision, avec des termes comme بصر
  • celui social, avec des mots comme النساء

Concrètement, « نظر » se situe à la fois dans un sens perceptif (vision/regard) et discursif (point de vue).

Pour les nuages de mots, la concaténation des corpus du dump et des corpus de contextes a permis d’obtenir 2 nuages différents. La concaténation s’est faite grâce aux formules suivantes :

Pour les dumps :

cat dump-text/ar-*.txt > nuages/dump-ar.txt

Pour les contextes :

cat contextes/ar-*.txt > nuages/contextes-ar.txt

Ensuite, nous avons généré les nuages avec wordcloud (dans un environnement virtuel) :

python3 nuages/wordcloud_ar.py
