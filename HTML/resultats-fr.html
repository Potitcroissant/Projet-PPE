Français

Dans le cas du français, pour pouvoir analyser les contextes dans lesquels le mot "regard" est utilisé, nous avons aussi utilisé le script PALS. Et fais les mêmes démarches que l’anglais et l’arabe. Pour éviter les répétitions, nous allons simplement expliquer les modifications effectuées et les quelques différences avec les scripts précédents.

Le code pour la tokénisation est donc le suivant, ce qui a été modifié sera en gras :

mot="[Rr]egard"
while read -r line
do
cat "dump-text/fr-${lineno}.txt" | sed -E 's/([^[:space:][:punct:]]+|[[:punct:]])/\1\n/g' | grep -v '^[[:space:]]*$' > "corpus-texte/fr/fr-$lineno.txt"
ligne=$(expr $ligne + 1) done < $fichier_urls

L’utilisations du scripts PALS varie seulement au niveau des chemins utilisés :

python3 PALS/cooccurrents.py --target [Rr]egard --match-mode regex corpus-texte/fr/* > resultat.csv

Le terminal nous indique directement que la regex [Rr]egard du mot apparaît 1622 fois, soit 1476 fois en minuscule et 146 fois avec la lettre majuscule.

En analysant les cooccurrences, on se rend compte que le mot "regard" apparaît dans divers contextes :

  • celui du genre, avec les mots "masculin" (104 fois) et "féminin" (85 fois)
  • celui sociologique, avec les mots "oppositionnel" (23 fois), "postcolonial" (22 fois) et "autrui" (36 fois)
  • celui psychologique, avec les mots "adolescent" (19 fois), "Enfances" (9 fois), "psy" (10 fois) et "contact" (30 fois)
  • celui de l’action, avec les mots "modifier" (75 fois), "porté" (26 fois), "basculer" (12 fois) et "tue" (12 fois)

Concrètement, le cadre contextuel dominant est celui du genre.

Pour les nuages de mots, la concaténation des corpus du dump et des corpus du contextes a permis d’obtenir 2 nuages différents. La concaténation s’est faite grâce à la formule suivante :

Pour les contextes :

cat *fr*.txt > corpus_en_un_contextes.txt

Pour les dumps :

cat *fr*.txt > corpus_en_un_dump.txt

Après cela, j’ai quasiment utilisé les mêmes formules que mes camarades mais en y ajoutant une touche d’esthétisme grâce à ces arguments :

  • --background white (fond d’écran)
  • --colormap YlOrBr (palette de couleur)
  • --contour_color gold (couleur de bordure)

Enfin, j’ai écrit les commandes pour les deux nuages en y ajoutant l’argument –stopword qui m’a permis d' exclure les mots vides de sens qui viendraient fausser l’analyse selon moi.

Attention, il faut savoir que dans mon cas, l’utilisation de wordcloud est possible grâce à l’environnement numérique dans lequel la bibliothèque wordcloud est installée. Pour ouvrir mon environnement, j’utilise la formule suivante :

source home/venvs/plurital/bin/activate
